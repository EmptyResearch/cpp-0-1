Lesson 0.1: Setting up a C++ Source File

What is C++?

    C++ is a low-level programming language, usually used for simple input/output and object-oriented programs. Though it was designed for small tasks, you can import images and more complex functions with external libraries (I use OpenCV). In this lesson, you will learn how to set up a basic C++ file, download a compiler, and export/save an exe on your computer.

Downloading a Compiler

    If you want to download a compiler, but you don't have a specific compiler in mind, then you should read this section. Otherwise, you can skip it.

    I like to sort compilers into three main categories: online, basic offline, and advanced offline compilers. If you are new to C++, and you don't need to export an exe for your projects, then try C++ Shell. It's a very basic, online compiler that compiles almost instantly. However, you can only create single-file C++ programs, and you can't export an exe.

    When you think you're ready for something more than C++ Shell, and you want a debugger, the ability to export an exe, and you want to create C++ projects with multiple files, try Quincy. Though a very simple IDE, it has a debugger, a builder (for exporting exes) , and it lets you create multiple files for one C++ project. It may look a little B-grade, but it does the job well.

    If you're ready to move on to the next level, which even I am not, then I recommend getting Visual Studio. I don't know much about Visual Studio, but I do know that it was created by Microsoft, and it builds (creates an exe and runs the program) extremely fast.

Setting up a C++ File

    Every C++ File stars with an #include <something> statement. Each statement does not end with a semicolon (;), and must be written on its own line. The most vital #include statement you will see (it's pretty much in every file) is #include <iostream>. It's an abbreviation for "input output stream", and it provides four very useful functions: std::cout, std::cin, std::cerr, std::clog.

    You might be wondering why they all start with "std::". This called the standard namespace, and most of the time you use these four functions, you'll be using std:: before each one. That's why you might see this phrase in a lot of C++ files: using namespace std;. This statement makes sure that all functions are "std::function", so you don't have to keep typing std:: all the time. The only problem with this statement is that every time you have to use a function that is in your own namespace (discussed later), you have to type using namespace myNamespace; myFunction; using namespace std; to switch namespaces.

    The last vital statement in a C++ file is int main(){}. Everything going on inside a program (besides defining variables, functions, arrays, etc.) must be inside that {}s of int main(){}; We'll discuss it in more detail later when we learn about functions.

Summary

  • Use a compiler that's right for you (basic: C++ Shell, intermediate: Quincy, advanced: Visual Studio).
  • Include #include <iostream> in almost every program you write.
  • Include using namespace std; inside of int main, but not in any other functions.
  • Most code should be inside int main() {put your code here} , except for function definition, and #include statements.
